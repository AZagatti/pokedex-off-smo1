# Build journal

- 2026-07-10: Starting build. Empty dir with only SPEC.md. Node v24.17.0, npm 11.17.0, git 2.43.0, gh CLI present.
- 2026-07-10: SvelteKit's new-style config has no `svelte.config.js` at all in this scaffolder version (kit 2.69) — adapter/paths/prerender all live inline in the `sveltekit({...})` vite plugin options. Not what I expected from docs memory; confirmed via `KitConfig & Options` type signature in `@sveltejs/kit/vite`.
- 2026-07-10: `npx ultracite init --hooks lefthook` crashes (`Error: Hook integration "lefthook" not found` — ultracite only knows a fixed set of hook managers, lefthook isn't one). Dropped `--hooks` and wrote `lefthook.yml` by hand instead.
- 2026-07-10: The oxlint config ultracite generated (`--frameworks sveltekit`) imports `ultracite/oxlint/sveltekit`, which doesn't exist in the installed package (only `ultracite/oxlint/svelte`). Had to hand-fix the import — looks like a bug/mismatch in ultracite's CLI vs published package for this version.
- 2026-07-10: ultracite's svelte oxlint preset is currently an empty stub (no rule overrides), so `unicorn/filename-case` (kebab-case) fired on `Header.svelte`. Added a local override in `oxlint.config.ts` disabling that rule for `**/*.svelte` — PascalCase filenames are the universal Svelte ecosystem convention and ultracite's preset just hasn't caught up yet.
- 2026-07-10: First e2e run passed against the WRONG app — `reuseExistingServer` (default true locally) picked up an unrelated `serve build -l 4173 -s` process already squatting on port 4173 from a different, unrelated parallel agent session on this same machine (saw `pokedex-glm-52-max` in its command line via `ps aux`). Playwright happily hit that other app and the assertion failed against its DOM. Fixed by moving this project to a dedicated port (4319) and forcing `reuseExistingServer: false` so this project's tests can never silently attach to another session's server. Worth remembering: this box runs multiple concurrent Pokedex-build agent sessions, so any shared default port/resource is a collision risk.
